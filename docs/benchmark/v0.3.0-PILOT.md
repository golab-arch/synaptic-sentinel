> 🚨 **PILOT — NOT a quality benchmark**. This is the **first real run** of the cross-provider benchmark (DG-076 plumbing) against 5 cloud providers + Ollama. It was preserved as evidence of **what the benchmark plumbing actually discovered** in its first contact with reality. The reporting numbers below should NOT be interpreted as "quality of each provider"; they are **the diagnostic surface that triggered DG-077 (post-mortem + surgical fixes + ground-truth recalibration)**. A second run with all fixes applied lives at `v0.3.0.md`.
>
> **Four findings exposed by this PILOT** (all addressed in DG-077):
>
> 1. **OpenAI gpt-5-nano: adapter bug**. 78/78 errors with `Unsupported parameter: 'max_tokens' is not supported with this model. Use 'max_completion_tokens' instead.` The `OpenAiCompatibleLlmClient` sends `max_tokens` to all OpenAI-compat backends; `gpt-5*` requires `max_completion_tokens`. Adapter needs per-model branch.
> 2. **DeepSeek deepseek-v3.2: model obsolete**. 78/78 errors with `The supported API model names are deepseek-v4-pro or deepseek-v4-flash, but you passed deepseek-v3.2.` Configuration default in `run.ts` + `pricing.json` is outdated.
> 3. **Gemini gemini-2.5-flash: JSON adherence broken via OpenAI-compat layer**. 77/78 errors mostly `The LLM response does not contain a JSON object`. Gemini wraps responses in markdown code blocks (` ```json\n...\n``` `) by default; the OpenAI-compat layer does NOT propagate `response_format: json_object` to Gemini's native `responseMimeType` param. Adapter needs Gemini-specific path.
> 4. **Anthropic Haiku 4.5: 100% JSON validity, 1.3% PASS — ground-truth bias**. Zero technical errors, but PASS rate atroz on a model that the original prompts (DG-024/DG-028/DG-033) were designed against. Forensic interpretation: the `requiredKeywords` of the DG-075 `ai-draft` ground truth were **picked from "what Claude (the author) would write"**, not from what real LLM responses contain. Example: keyword `["eval", "code injection"]` literal; real responses say `eval` + `RCE` or `arbitrary code execution` — synonyms that fail substring containment. **This confirms the DG-075 anti-optimismo caveat verbatim**.
>
